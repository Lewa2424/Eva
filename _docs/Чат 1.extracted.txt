Да, как формулировка плана — это ок и значительно сильнее, чем исходный 1.4. Ты превращаешь “установить Docker” в контур безопасности, без которого самомодификация — русская рулетка.
Но есть 3 критических поправки, иначе в документе появятся ложные гарантии и риск “дыру не заметили”.
1) Убери обещание “гарантирует 100% локализацию”
Docker ≠ абсолютная изоляция. Это изоляция уровня ОС, но уязвимости, ошибки конфигурации, неправильные монтирования, --privileged, доступ к Docker socket, kernel-bugs — всё это ломает “100%”.
Корректнее так:
“максимально локализует риски в пределах модели угроз MVP”
“снижает вероятность воздействия на хост до приемлемого уровня при соблюдении политик”
Иначе ты сам себе создаёшь юридически/логически неверный якорь.
2) “Отключение сети по умолчанию” — правильно, но добавь режимы
В реальности часть задач потребует сети (pip install, загрузка моделей/данных). Поэтому надо зафиксировать в плане режимы песочницы:
SAFE (default): --network=none
FETCH: сеть разрешена только на время установки зависимостей (и лучше с allowlist-доменами на будущее, хотя на MVP можно просто кратковременно)
ONLINE: редкий режим, только по явному разрешению
Это важно, чтобы DecisionEngine позже мог выбирать режим сам, а не “или всегда без сети, или всегда с дырой”.
3) Seccomp — хорошо, но не обещай “строгие политики” без уточнения
Seccomp-профили реально усиливают безопасность, но:
“строгий” профиль может ломать вполне нормальные вещи (pytest, компиляции, некоторые библиотеки).
на MVP разумнее написать:“использовать стандартный seccomp-профиль Docker + возможность переключения на строгий профиль для задач с повышенным риском”.

- Запрет эскалации: не использовать --privileged, не монтировать /var/run/docker.sock и устройства хоста (/dev/*).

Цель:

- Успешный запуск контейнера с выполнением тривиального скрипта (print("Hello")).

Теперь — строго и по фактам.
1. Ограничение в ~500 строк — технически корректно
Почему это работает:
Python не имеет ограничений на количество файлов или модулей.
Импорты и разбиение логики никак не страдают при дроблении.
Большие проекты (Django, FastAPI, Kubernetes-клиенты) живут именно так: много маленьких файлов, а не “монолиты”.
500 строк — это:
разумный верхний предел читаемости;
комфортный объём для анализа AST;
безопасный размер для передачи в LLM без “утопления контекста”.
2. Для самомодифицирующегося ИИ это критически важно
Ты интуитивно попал в фундаментальный принцип self-modifying systems:
ИИ должен иметь возможность “взять в голову” целый модуль целиком.
Если файл:
200–400 строк → Ева может понять его полностью, а не по фрагментам;
1000 строк → начинаются “слепые зоны”, частичные правки, регрессии.
Для ИИ это даже важнее, чем для человека.
3. Как это оформить правильно (чтобы не превратилось в хаос)
Жёсткое правило:
Один файл = одна роль.
Если файл:
растёт >400–500 строк,
содержит 2 логически разных процесса,
→ делить без обсуждений.
Пример (как должно быть):
eva_core/
├─ decision_engine.py # только координация
├─ goal_handler.py # работа с целями
├─ action_router.py # маршрутизация действий
├─ lifecycle.py # запуск / остановка / цикл
А не:
decision_engine.py (1200 строк ада)
4. Это напрямую улучшает “синапсы”
Маленькие файлы дают:
чёткие входы/выходы;
минимальные зависимости;
возможность переподключать модули, не ломая всё.
То есть:
файл = нейронный узел;
импорт = синапс.
Это полностью соответствует твоей биологической метафоре — и не в поэзии, а в реальной инженерии.
5. Единственное предупреждение (и оно важное)
Не дроби раньше времени.
Плохой вариант:
40 файлов по 20 строк “на будущее”.
Правильный вариант:

Что DecisionCore ЕСТЬ
DecisionCore = оркестратор, который:
не выполняет работу сам;
не генерирует код;
не тестирует;
не лезет в детали реализации.
Он:
принимает цель;
решает что делать дальше;
выбирает какой отдел активировать;
отслеживает ход выполнения;
принимает решение “достаточно / повторить / откатить”.
Это ровно роль дирижёра:
он не играет на скрипке,
но без него оркестр — шум.
Что DecisionCore НЕ ДОЛЖЕН делать (жёстко)
Если DecisionCore:
содержит бизнес-логику генерации,
знает, как править код,
знает, как запускать Docker,
содержит SQL-запросы,
→ архитектура сломана.
Все “как” — вне его.

DecisionCore:

LLMProxy — единая точка доступа к языковым моделям (LLM) для Евы 2.0.
Зачем нужен:
- Скрывает детали SDK OpenAI и переменных окружения.
- Даёт единый интерфейс: "сделай запрос" -> "получи ответ".
- Маршрутизирует запросы по моделям:
- OPENAI_CHAT_MODEL (например, gpt-4o) для диалога/планов/объяснений
- OPENAI_CODE_MODEL (например, gpt-5.2) строго для кода/тестов/рефакторинга
Правило архитектуры:
- В других модулях НЕ должно быть прямых вызовов OpenAI SDK.
- Только через LLMProxy (одна дверь = контроль и безопасность).
"""
import os
from dataclasses import dataclass
from dotenv import load_dotenv
@dataclass(frozen=True)
class LLMModels:
"""
Контейнер имён моделей.
Зачем:
- Явно фиксируем, какие модели используются.
- Упрощаем самодиагностику и будущую замену моделей.
"""
chat: str
code: str
class LLMProxy:
"""
Прокси для запросов к LLM.
На этом шаге — каркас (интерфейс + выбор модели).
Реальные сетевые вызовы добавим позже, когда закрепим политику и формат промптов.
"""
def __init__(self):
# Что делаем:
# - Загружаем .env (если он есть).
# Зачем:
# - Чтобы конфигурация (ключи/модели) не хранилась в коде.
load_dotenv()
# Что берём:
# - Ключ (позже потребуется для реальных запросов).
self.api_key = os.getenv("OPENAI_API_KEY", "").strip()
# Что берём:
# - Имена моделей для двух режимов работы.
self.models = LLMModels(
chat=os.getenv("OPENAI_CHAT_MODEL", "gpt-4o").strip(),
code=os.getenv("OPENAI_CODE_MODEL", "gpt-5.2").strip(),
)
def choose_model(self, task_type: str) -> str:
"""
Что делает:
- Выбирает модель по типу задачи.
task_type:
- "chat" -> модель диалога/планирования
- "code" -> модель кода/тестов
Зачем:
- DecisionEngine и остальные модули дают только тип задачи,
а выбор модели централизован.
"""
task_type = (task_type or "").strip().lower()
if task_type == "code":
return self.models.code
# По умолчанию — "chat"
return self.models.chat
def request(self, task_type: str, prompt: str) -> str:
"""
Что делает:
- Единая точка запроса к LLM.
Сейчас:
- Возвращает заглушку (stub), чтобы остальные модули могли уже подключаться
без сетевых вызовов.
Позже:
- Здесь будет реальный вызов OpenAI SDK с политиками безопасности.
"""
model = self.choose_model(task_type=task_type)
# Заглушка. В будущем заменим на реальный ответ модели.
return f"[LLM STUB] model={model} prompt_len={len(prompt)}"

-- Задачи, полученные Евой
CREATE TABLE IF NOT EXISTS tasks (
id SERIAL PRIMARY KEY,
goal TEXT NOT NULL,
status TEXT NOT NULL DEFAULT 'new',
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
-- Планы выполнения задач
CREATE TABLE IF NOT EXISTS plans (
id SERIAL PRIMARY KEY,
task_id INTEGER REFERENCES tasks(id) ON DELETE CASCADE,
plan TEXT NOT NULL,
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
-- Версии кода и патчи
CREATE TABLE IF NOT EXISTS code_versions (
id SERIAL PRIMARY KEY,
module_name TEXT NOT NULL,
code TEXT NOT NULL,
meta JSONB,
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
-- Результаты тестов
CREATE TABLE IF NOT EXISTS test_results (
id SERIAL PRIMARY KEY,
task_id INTEGER REFERENCES tasks(id) ON DELETE CASCADE,
status TEXT NOT NULL,
output TEXT,
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
-- Рефлексия и выводы
CREATE TABLE IF NOT EXISTS reflections (
id SERIAL PRIMARY KEY,
task_id INTEGER REFERENCES tasks(id) ON DELETE CASCADE,
reflection TEXT NOT NULL,
meta JSONB,
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- ============================================================
-- ЕВА 2.0 — ПАМЯТЬ (SCHEMA v1 / MVP+)
-- Без векторизации (pgvector добавим позже миграцией)
-- ============================================================
-- 1) Задачи
CREATE TABLE IF NOT EXISTS tasks (
id BIGSERIAL PRIMARY KEY,
goal TEXT NOT NULL,
status TEXT NOT NULL DEFAULT 'new',
source TEXT NOT NULL DEFAULT 'user', -- user | self | system
meta JSONB,
created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
-- 2) Планы (привязаны к задаче)
CREATE TABLE IF NOT EXISTS plans (
id BIGSERIAL PRIMARY KEY,
task_id BIGINT NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
plan_text TEXT NOT NULL,
meta JSONB,
created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
-- 3) Версии кода (привязаны к модулю/файлу)
CREATE TABLE IF NOT EXISTS code_versions (
id BIGSERIAL PRIMARY KEY,
module_name TEXT NOT NULL,
status TEXT NOT NULL DEFAULT 'draft', -- draft | tested | active | deprecated
code_text TEXT NOT NULL,
meta JSONB,
created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
-- 4) Результаты тестов (привязаны к задаче + опционально к версии кода)
CREATE TABLE IF NOT EXISTS test_results (
id BIGSERIAL PRIMARY KEY,
task_id BIGINT NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
code_version_id BIGINT REFERENCES code_versions(id) ON DELETE SET NULL,
status TEXT NOT NULL, -- pass | fail | error
output TEXT,
meta JSONB,
created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
-- 5) Рефлексия (уроки/выводы, привязаны к задаче)
CREATE TABLE IF NOT EXISTS reflections (
id BIGSERIAL PRIMARY KEY,
task_id BIGINT NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
reflection TEXT NOT NULL,
meta JSONB,
created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
-- =========================
-- ИНДЕКСЫ (обязательная база)
-- =========================
CREATE INDEX IF NOT EXISTS idx_tasks_status_created
ON tasks(status, created_at DESC);
CREATE INDEX IF NOT EXISTS idx_plans_task_created
ON plans(task_id, created_at DESC);
CREATE INDEX IF NOT EXISTS idx_code_versions_module_created
ON code_versions(module_name, created_at DESC);
CREATE INDEX IF NOT EXISTS idx_test_results_task_created
ON test_results(task_id, created_at DESC);
CREATE INDEX IF NOT EXISTS idx_reflections_task_created
ON reflections(task_id, created_at DESC);

MemoryManager — долговременная память Евы 2.0 (PostgreSQL).

from __future__ import annotations
import os
from typing import Any, Optional
import psycopg2
from psycopg2.extras import RealDictCursor
from dotenv import load_dotenv
class MemoryManager:
def __init__(self) -> None:
load_dotenv()
dsn = (os.getenv("POSTGRES_DSN") or "").strip()
if not dsn:
raise RuntimeError("POSTGRES_DSN is not set. Add it to .env")
self._conn = psycopg2.connect(dsn)
self._conn.autocommit = True
def close(self) -> None:
"""Закрывает соединение с БД."""
if getattr(self, "_conn", None):
self._conn.close()
def ping(self) -> bool:
"""Проверяет, что соединение с БД живое."""
with self._conn.cursor() as cur:
cur.execute("SELECT 1;")
return cur.fetchone() == (1,)
# -------------------------

# -------------------------
def get_context_for_goal(self, goal: str, limit: int = 10) -> dict:
"""
Возвращает контекст для цели:
- последние задачи
- последние планы по ним
- последние рефлексии по ним
"""
context: dict[str, Any] = {"goal": goal, "recent_tasks": [], "plans": [], "reflections": []}
with self._conn.cursor(cursor_factory=RealDictCursor) as cur:
cur.execute(
"""
SELECT id, goal, status, source, meta, created_at, updated_at
FROM tasks
ORDER BY created_at DESC
LIMIT %s;
""",
(limit,),
)
tasks = cur.fetchall()
context["recent_tasks"] = tasks
task_ids = [t["id"] for t in tasks]
if task_ids:
cur.execute(
"""
SELECT id, task_id, plan_text, meta, created_at
FROM plans
WHERE task_id = ANY(%s)
ORDER BY created_at DESC
LIMIT %s;
""",
(task_ids, limit),
)
context["plans"] = cur.fetchall()
cur.execute(
"""
SELECT id, task_id, reflection, meta, created_at
FROM reflections
WHERE task_id = ANY(%s)
ORDER BY created_at DESC
LIMIT %s;
""",
(task_ids, limit),
)
context["reflections"] = cur.fetchall()
return context
def get_module_info(self, module_name: str) -> dict:
"""
Возвращает информацию о модуле:
- последняя версия кода
- количество версий
- последняя дата изменения
"""
module_name = (module_name or "").strip()
if not module_name:
return {"module_name": "", "latest": None, "versions_count": 0}
with self._conn.cursor(cursor_factory=RealDictCursor) as cur:
cur.execute(
"""
SELECT id, module_name, status, meta, created_at
FROM code_versions
WHERE module_name = %s
ORDER BY created_at DESC
LIMIT 1;
""",
(module_name,),
)
latest = cur.fetchone()
cur.execute(
"""
SELECT COUNT(*) AS cnt
FROM code_versions
WHERE module_name = %s;
""",
(module_name,),
)
cnt = cur.fetchone()["cnt"]
return {"module_name": module_name, "latest": latest, "versions_count": cnt}
def get_recent_errors(self, module_name: str, limit: int = 10) -> list[dict]:
"""
Возвращает последние неуспешные результаты тестов по указанному модулю.
Связь идёт через code_versions (module_name) -> test_results(code_version_id).
"""
module_name = (module_name or "").strip()
if not module_name:
return []
with self._conn.cursor(cursor_factory=RealDictCursor) as cur:

SELECT tr.id,
tr.task_id,
tr.code_version_id,
tr.status,
tr.output,
tr.meta,
tr.created_at
FROM test_results tr
JOIN code_versions cv ON cv.id = tr.code_version_id
WHERE cv.module_name = %s
AND tr.status <> 'pass'
ORDER BY tr.created_at DESC
LIMIT %s;
""",
(module_name, limit),
)
return cur.fetchall()

файл исключён из поиска / индексирования(обычно через .gitignore или настройки Explorer)
Это НЕ означает:
что файл не существует ❌
что Python его не видит ❌
что dotenv его не читает ❌
Это только UI-поведение VS Code.
Важно
python-dotenv читает .env напрямую по имени,ему плевать на глазик в VS Code.
2. Главное: у тебя сейчас .env = .env.example
По содержимому, которое ты прислал, видно ключевое:
OPENAI_API_KEY=__PUT_YOUR_KEY_HERE__
POSTGRES_DSN=postgresql://USER:PASSWORD@HOST:5432/DBNAME
❗ Это НЕ рабочий .env, это всё ещё шаблон.Даже если файл называется .env.
Что это означает
MemoryManager упадёт, потому что:
DSN невалидный
LLMProxy не сможет работать, потому что:
ключ — заглушка
3. Что нужно сделать СЕЙЧАС (аккуратно, без спешки)
Шаг 1. Оставляем .env.example как есть
Его не трогаем. Это эталон/шаблон.
Шаг 2. Правим ТОЛЬКО .env
Открой .env и замени ТОЛЬКО значения, комментарии оставь.
Минимум, который ОБЯЗАТЕЛЕН для текущего шага:
OPENAI_API_KEY=твой_реальный_ключ_здесь
OPENAI_CHAT_MODEL=gpt-4o
OPENAI_CODE_MODEL=gpt-5.2
POSTGRES_DSN=postgresql://eva_user:ТВОЙ_ПАРОЛЬ@localhost:5432/Eva_2.0
SANDBOX_MODE=SAFE
EVA_MODE=development
MAX_FILE_LINES=500
🔹 eva_user — тот самый пользователь, которого ты создал🔹 ТВОЙ_ПАРОЛЬ — пароль, который ты задал ему🔹 Eva_2.0 — строго как в pgAdmin, регистр важен
4. Проверка, что .env реально читается
Сделай одну строку в Python (можно прямо в REPL):
from dotenv import load_dotenv; import os; load_dotenv(); print(os.getenv("POSTGRES_DSN"))
Если ты видишь строку подключения, а не None — всё ок.

VS Code защищает от случайного коммита секретов
